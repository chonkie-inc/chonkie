---
title: "Recursive Chunker"
api: "POST /v1/chunk/recursive"
description: "Hierarchically chunk text using multiple delimiters for structured content"
---

The Recursive Chunker uses a hierarchical approach to split text, trying multiple delimiters in order until the desired chunk size is achieved. This is ideal for structured documents like markdown, code, or formatted text.

## Authentication

Set your API key as an environment variable or pass it to the SDK:

```bash
export CHONKIE_API_KEY="your-api-key-here"
```

Get your API key from [labs.chonkie.ai](https://labs.chonkie.ai)

## Request

#### Parameters

<ParamField path="text" type="string | string[]" required>
  The text to chunk.
</ParamField>

<ParamField path="tokenizer" type="string" default="gpt2">
  Tokenizer to use for counting tokens.
</ParamField>

<ParamField path="chunk_size" type="integer" default="512">
  Maximum number of tokens per chunk.
</ParamField>

<ParamField path="chunk_overlap" type="integer" default="0">
  Number of tokens to overlap between chunks.
</ParamField>

<ParamField path="min_characters_per_chunk" type="integer" default="24">
  Minimum character length for chunks.
</ParamField>

## Response

#### Returns

Array of `Chunk` objects with text, start_index, end_index, and token_count.

## Examples

<CodeGroup>

```python Python
from chonkie.cloud import RecursiveChunker

# Initialize with default rules (good for markdown/text)
chunker = RecursiveChunker(
    tokenizer="gpt2",
    chunk_size=512,
    chunk_overlap=50
)

# Chunk markdown content
text = """
# Machine Learning

Machine learning is a subset of artificial intelligence.

## Supervised Learning

In supervised learning, models learn from labeled data.

## Unsupervised Learning

Unsupervised learning finds patterns without labels.
"""

chunks = chunker.chunk(text)

for chunk in chunks:
    print(f"Chunk: {chunk.text[:80]}...")
    print(f"Tokens: {chunk.token_count}\n")
```

```javascript JavaScript
import { RecursiveChunker } from '@chonkiejs/cloud';

// Initialize with default rules
const chunker = new RecursiveChunker({
  tokenizer: 'gpt2',
  chunkSize: 512,
  chunkOverlap: 50
});

// Chunk markdown content
const text = `
# Machine Learning

Machine learning is a subset of artificial intelligence.

## Supervised Learning

In supervised learning, models learn from labeled data.
`;

const chunks = await chunker.chunk({ text });

for (const chunk of chunks) {
  console.log(`Chunk: ${chunk.text.substring(0, 80)}...`);
  console.log(`Tokens: ${chunk.tokenCount}\n`);
}
```

## How It Works

The Recursive Chunker tries delimiters in this order:

1. **Double newlines** (`\n\n`) - Paragraph breaks
2. **Single newlines** (`\n`) - Line breaks
3. **Periods** (`. `) - Sentence ends
4. **Spaces** (` `) - Word boundaries
5. **Characters** - Individual characters as last resort

This ensures the most natural split points are used first.

## Use Cases

- **Markdown Documents** - Preserve heading hierarchy and structure
- **Code Files** - Maintain logical code sections
- **Structured Text** - Keep formatting and organization intact
- **Mixed Content** - Handle documents with varied structure

## Best Practices

<Tip>
  The Recursive Chunker works best with well-structured documents that have clear hierarchical markers like headings, paragraphs, and sections.
</Tip>

<Tip>
  For code files, the recursive approach preserves function and class boundaries better than simple token chunking.
</Tip>

<Warning>
  Unstructured text without clear delimiters may not benefit much from recursive chunking. Consider using Sentence or Token chunker instead.
</Warning>
