---
title: "Create Agent"
api: "POST /v1/agents"
description: "Create an AI agent with custom configuration and knowledge base access"
---

Create a new AI agent with customizable LLM settings, system prompts, and knowledge base integration. Agents can be used for chat, task automation, and research.

## Authentication

Include your API key in the Authorization header:

```bash
Authorization: Bearer YOUR_API_KEY
```

Get your API key from [labs.chonkie.ai](https://labs.chonkie.ai)

## Request

#### Parameters

<ParamField path="name" type="string" required>
  Agent name (3-100 characters).
</ParamField>

<ParamField path="description" type="string">
  Description of the agent's purpose and capabilities.
</ParamField>

<ParamField path="model" type="string" default="gpt-4o-mini">
  LLM model to use. Options: "gpt-4o", "gpt-4o-mini", "gpt-4-turbo", "gpt-3.5-turbo", "groq/llama-3.1-70b", "groq/mixtral-8x7b".
</ParamField>

<ParamField path="systemPrompt" type="string">
  System prompt that defines the agent's behavior and personality.
</ParamField>

<ParamField path="temperature" type="float" default="0.7">
  Creativity level (0.0-2.0). Lower is more focused, higher is more creative.
</ParamField>

<ParamField path="maxContextChunks" type="integer" default="5">
  Maximum number of knowledge base chunks to include in context.
</ParamField>

<ParamField path="useContext" type="boolean" default="false">
  Whether to inject knowledge base context into conversations.
</ParamField>

<ParamField path="knowledge" type="string">
  Knowledge base slug to use for context (required if useContext is true).
</ParamField>

## Response

#### Returns

<ResponseField name="slug" type="string">
  Unique slug for the agent (used in API calls).
</ResponseField>

<ResponseField name="agent" type="object">
  Complete agent configuration.
</ResponseField>

## Examples

<CodeGroup>

```python Python
import requests

url = "https://labs.chonkie.ai/api/v1/agents"
headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}

data = {
    "name": "Documentation Assistant",
    "description": "Helps users navigate product documentation",
    "model": "gpt-4o-mini",
    "systemPrompt": "You are a helpful documentation assistant. Answer questions clearly and concisely based on the provided context. If you don't know the answer, say so.",
    "temperature": 0.3,
    "maxContextChunks": 5,
    "useContext": True,
    "knowledge": "product-documentation"
}

response = requests.post(url, headers=headers, json=data)
result = response.json()

print(f"Created agent: {result['agent']['name']}")
print(f"Slug: {result['slug']}")
print(f"Model: {result['agent']['model']}")
```

## Available Models

### OpenAI Models
- **gpt-4o** - Latest GPT-4 Omni (best quality, slower)
- **gpt-4o-mini** - Fast GPT-4 Omni (good quality, faster)
- **gpt-4-turbo** - GPT-4 Turbo (high quality)
- **gpt-3.5-turbo** - GPT-3.5 (fastest, lower cost)

### Groq Models (Ultra-fast)
- **groq/llama-3.1-70b** - Meta's Llama 3.1 70B
- **groq/mixtral-8x7b** - Mistral's Mixtral 8x7B

## Temperature Guide

Choose temperature based on use case:

| Temperature | Best For | Behavior |
|------------|----------|----------|
| 0.0 - 0.3 | Factual Q&A, documentation, analysis | Focused, deterministic, consistent |
| 0.4 - 0.7 | General chat, customer support | Balanced creativity and accuracy |
| 0.8 - 1.2 | Creative writing, brainstorming | More varied, creative responses |
| 1.3 - 2.0 | Experimental, highly creative tasks | Very creative, less predictable |

## Context Configuration

When `useContext` is true, the agent retrieves relevant chunks from the knowledge base for each query:

```python
# Context-enabled agent
agent = {
    "useContext": True,
    "knowledge": "product-documentation",
    "maxContextChunks": 5  # Top 5 most relevant chunks
}
```

The agent automatically:
1. Searches the knowledge base for relevant content
2. Injects top `maxContextChunks` into the conversation
3. Uses this context to answer questions accurately

## System Prompt Best Practices

Effective system prompts:

```python
# Good: Clear role and behavior
systemPrompt = """You are a helpful customer support agent.
- Answer questions clearly and concisely
- Be friendly and professional
- If unsure, admit it and offer to escalate
- Always cite specific documentation when possible"""

# Good: Specific domain expertise
systemPrompt = """You are an expert Python developer assistant.
- Provide code examples with explanations
- Follow PEP 8 style guidelines
- Suggest best practices and optimizations
- Explain complex concepts simply"""

# Avoid: Vague or overly long prompts
systemPrompt = "You help people."  # Too vague
```

## Best Practices

<Tip>
  Set `useContext=true` and link a knowledge base for agents that need to reference specific information. This enables RAG-powered responses.
</Tip>

<Tip>
  Use lower temperatures (0.2-0.4) for factual tasks, higher temperatures (0.7-0.9) for creative tasks.
</Tip>

<Tip>
  Start with `maxContextChunks=5`. Increase to 10-15 for complex queries that need more context, decrease to 3 for simpler tasks.
</Tip>

<Tip>
  Choose model based on your needs: gpt-4o-mini for most tasks, gpt-4o for complex reasoning, groq models for speed.
</Tip>

<Warning>
  Agents with `useContext=true` must have a valid `knowledge` field. The knowledge base must exist and contain indexed files.
</Warning>

## Error Responses

</ResponseField>

<ResponseField name="404 Not Found">
  Referenced knowledge base doesn't exist.
</ResponseField>

<ResponseField name="409 Conflict">
  Agent with this name already exists in your organization.
</ResponseField>

<ResponseField name="429 Too Many Requests">
  Rate limit exceeded or too many agents in organization.
</ResponseField>
