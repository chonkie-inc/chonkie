---
title: "Create Knowledge Base"
api: "POST /v1/knowledge"
description: "Create a new knowledge base for storing and indexing documents"
---

Create a new knowledge base that can store documents, generate embeddings, and enable semantic search. Each knowledge base gets its own storage bucket and vector collection.

## Authentication

Include your API key in the Authorization header:

```bash
Authorization: Bearer YOUR_API_KEY
```

Get your API key from [labs.chonkie.ai](https://labs.chonkie.ai)

## Request

#### Parameters

<ParamField path="name" type="string" required>
  Name of the knowledge base. Will be used to generate a unique slug.
</ParamField>

<ParamField path="description" type="string">
  Optional description of what this knowledge base contains.
</ParamField>

## Response

#### Returns

<ResponseField name="id" type="string">
  Unique identifier for the knowledge base.
</ResponseField>

<ResponseField name="slug" type="string">
  URL-safe slug generated from the name. Use this for subsequent API calls.
</ResponseField>

<ResponseField name="name" type="string">
  The knowledge base name.
</ResponseField>

<ResponseField name="description" type="string">
  The knowledge base description.
</ResponseField>

<ResponseField name="bucket" type="string">
  R2 bucket name for file storage.
</ResponseField>

<ResponseField name="collection" type="string">
  Qdrant collection name for vector storage.
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp of creation.
</ResponseField>

## Examples

<CodeGroup>

```python Python
import requests

url = "https://labs.chonkie.ai/api/v1/knowledge"
headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}

data = {
    "name": "Product Documentation",
    "description": "Technical documentation for our products"
}

response = requests.post(url, headers=headers, json=data)
kb = response.json()

print(f"Created knowledge base: {kb['name']}")
print(f"Slug: {kb['slug']}")
print(f"ID: {kb['id']}")
```

```javascript JavaScript
const response = await fetch('https://labs.chonkie.ai/api/v1/knowledge', {
  method: 'POST',

  },
  body: JSON.stringify({

  })
});

const kb = await response.json();
console.log(`Created knowledge base: ${kb.name}`);
console.log(`Slug: ${kb.slug}`);
console.log(`ID: ${kb.id}`);
```

## What Gets Created

When you create a knowledge base, Chonkie automatically provisions:

1. **R2 Storage Bucket** - For storing your original files
2. **Qdrant Vector Collection** - For semantic search over embeddings
3. **Database Entry** - Metadata and configuration
4. **Unique Slug** - URL-safe identifier for API calls

## Slug Generation

The slug is automatically generated from the name:

- **Lowercase** - "Product Docs" → "product-docs"
- **Spaces to hyphens** - "My Knowledge Base" → "my-knowledge-base"
- **Special chars removed** - "Docs (v2.0)" → "docs-v20"
- **Unique** - If slug exists, a suffix is added

## Best Practices

<Tip>
  Use descriptive names that reflect the content. The name becomes part of the slug, making API calls more readable.
</Tip>

<Tip>
  Add a detailed description to help you remember what each knowledge base contains, especially when managing multiple bases.
</Tip>

<Info>
  Knowledge bases are scoped to your organization. Team members can access all knowledge bases in your organization.
</Info>

<Warning>
  Knowledge base names must be unique within your organization. Choose distinct names to avoid confusion.
</Warning>

## Limits

- **Name length**: 3-100 characters
- **Description length**: Up to 500 characters
- **Knowledge bases per organization**: Check your plan limits at [labs.chonkie.ai](https://labs.chonkie.ai)
