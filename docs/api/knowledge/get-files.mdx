---
title: "Get Knowledge Base Files"
api: "GET /v1/knowledge/{knowledge_base_slug}"
description: "List all files in a knowledge base"
---

Retrieve a list of all files stored in a specific knowledge base, including metadata like file size, upload date, and file type.

## Authentication

Include your API key in the Authorization header:

```bash
Authorization: Bearer YOUR_API_KEY
```

Get your API key from [labs.chonkie.ai](https://labs.chonkie.ai)

## Request

#### Path Parameters

<ParamField path="knowledge_base_slug" type="string" required>
  The slug of the knowledge base to query. Get this from [List Knowledge
  Bases](/api/knowledge/list) or [Create Knowledge Base](/api/knowledge/create).
</ParamField>

## Response

#### Returns

<ResponseField name="files" type="array">
  Array of file objects in the knowledge base.
</ResponseField>

Each file object contains:

<ResponseField name="key" type="string">
  Unique file identifier (format: timestamp-filename).
</ResponseField>

<ResponseField name="name" type="string">
  Original filename.
</ResponseField>

<ResponseField name="size" type="integer">
  File size in bytes.
</ResponseField>

<ResponseField name="lastModified" type="string">
  ISO 8601 timestamp of last modification.
</ResponseField>

<ResponseField name="contentType" type="string">
  MIME type of the file.
</ResponseField>

## Examples

<CodeGroup>

```python Python
import requests

kb_slug = "product-documentation"
url = f"https://labs.chonkie.ai/api/v1/knowledge/{kb_slug}"
headers = {
    "Authorization": "Bearer YOUR_API_KEY"
}

response = requests.get(url, headers=headers)
data = response.json()
files = data["files"]

print(f"Files in '{kb_slug}':\n")
for file in files:
    size_kb = file["size"] / 1024
    print(f"- {file['name']}")
    print(f"  Size: {size_kb:.2f} KB")
    print(f"  Type: {file['contentType']}")
    print(f"  Uploaded: {file['lastModified']}\n")
```

```javascript JavaScript
const kbSlug = "product-documentation";
const response = await fetch(
  `https://labs.chonkie.ai/api/v1/knowledge/${kbSlug}`,
  {
    headers: {
      Authorization: "Bearer YOUR_API_KEY",
    },
  }
);

const { files } = await response.json();

console.log(`Files in '${kbSlug}':\n`);
files.forEach((file) => {
  const sizeKB = (file.size / 1024).toFixed(2);
  console.log(`- ${file.name}`);
  console.log(`  Size: ${sizeKB} KB`);
  console.log(`  Type: ${file.contentType}`);
  console.log(`  Uploaded: ${file.lastModified}\n`);
});
```

```bash cURL
curl -X GET https://labs.chonkie.ai/api/v1/knowledge/product-documentation \
  -H "Authorization: Bearer YOUR_API_KEY"
```

</CodeGroup>

## File Key Format

The `key` field uses the format: `{timestamp}-{filename}`

- **Timestamp** - Unix millisecond timestamp of upload
- **Filename** - Original filename

Use the full key when calling [Delete File](/api/knowledge/delete-file) or [Download File](/api/knowledge/download).

## Supported File Types

Common file types and their MIME types:

| Extension       | MIME Type          | Description    |
| --------------- | ------------------ | -------------- |
| `.pdf`          | application/pdf    | PDF documents  |
| `.txt`          | text/plain         | Plain text     |
| `.md`           | text/markdown      | Markdown       |
| `.doc`, `.docx` | application/msword | Word documents |
| `.html`         | text/html          | HTML files     |
| `.json`         | application/json   | JSON files     |
| `.csv`          | text/csv           | CSV files      |

## Use Cases

- **File Management** - Display uploaded files in your UI
- **Storage Monitoring** - Track storage usage
- **Cleanup** - Identify old or large files for removal
- **Status Checking** - Verify file uploads completed successfully

## Best Practices

<Tip>
  Store the file `key` in your database if you need to reference specific files
  later for deletion or download.
</Tip>

<Tip>
  Display file sizes in human-readable format (KB, MB) for better user
  experience.
</Tip>

<Info>
  Files are listed in reverse chronological order (newest first) by default.
</Info>

<Warning>
  Very large knowledge bases with thousands of files may take longer to list.
  Consider implementing pagination in your UI.
</Warning>

## Empty Knowledge Base

If a knowledge base has no files, the response will be:

```json
{
  "files": [],
  "knowledgeBase": {
    "id": "kb_abc123def456",
    "slug": "product-documentation",
    "name": "Product Documentation"
  }
}
```

</ResponseField>
