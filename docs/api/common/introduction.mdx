Welcome to Chonkie's Cloud API! Build powerful AI applications with enterprise-grade text chunking, semantic search, and intelligent agents.

## What is Chonkie Cloud?

Chonkie Cloud provides two complementary APIs:

### ðŸ”ª Chunking & Embedding API
**Base URL**: `https://api.chonkie.ai`

Transform raw text into perfectly-sized, semantically meaningful chunks for RAG applications. Includes:

- **8 Advanced Chunkers** - Token, Sentence, Recursive, Semantic, Late, Code, Neural, and Slumber
- **2 Refineries** - Add embeddings and overlap for better retrieval
- **Multiple Embeddings** - Support for all major embedding models
- **Battle-tested** - Used in production by thousands of developers

### ðŸ§  Context as a Service API
**Base URL**: `https://labs.chonkie.ai/api`

Build AI applications with persistent knowledge and intelligent agents:

- **Knowledge Bases** - Store and index documents with automatic chunking and embedding
- **Semantic Search** - Three-mode search (fast/balanced/deep) over your documents
- **AI Agents** - Configurable agents with LLM integration and RAG capabilities
- **MCP Integration** - Connect to Claude Desktop via Model Context Protocol

## Quick Start

### 1. Get Your API Key

Sign up at [labs.chonkie.ai](https://labs.chonkie.ai) to get your API key.

### 2. Install SDK (Optional)

<CodeGroup>

```bash Python
pip install chonkie[cloud]
```

```bash JavaScript
npm install @chonkiejs/cloud
```

</CodeGroup>

### 3. Make Your First Request

<CodeGroup>

```python Python - Chunking
from chonkie.cloud import TokenChunker

# Initialize chunker
chunker = TokenChunker(
    chunk_size=512,
    chunk_overlap=50
)

# Chunk your text
text = "Your long document here..."
chunks = chunker.chunk(text)

print(f"Created {len(chunks)} chunks")
```

```javascript JavaScript - Chunking
import { TokenChunker } from '@chonkiejs/cloud';

// Initialize chunker
const chunker = new TokenChunker({
  chunkSize: 512,
  chunkOverlap: 50
});

// Chunk your text
const text = "Your long document here...";
const chunks = await chunker.chunk({ text });

console.log(`Created ${chunks.length} chunks`);
```

```python Python - Knowledge Base
import requests

# Create knowledge base
response = requests.post(
    "https://labs.chonkie.ai/api/v1/knowledge",
    headers={
        "Authorization": "Bearer YOUR_API_KEY",
        "Content-Type": "application/json"
    },
    json={
        "name": "My Docs",
        "description": "Product documentation"
    }
)

kb = response.json()
print(f"Created: {kb['slug']}")
```

```javascript JavaScript - Knowledge Base
// Create knowledge base
const response = await fetch(
  'https://labs.chonkie.ai/api/v1/knowledge',
  {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      name: 'My Docs',
      description: 'Product documentation'
    })
  }
);

const kb = await response.json();
console.log(`Created: ${kb.slug}`);
```

</CodeGroup>

## Authentication

All API requests require authentication using Bearer tokens:

```bash
Authorization: Bearer YOUR_API_KEY
```

Set it as an environment variable for convenience:

```bash
export CHONKIE_API_KEY="your-api-key-here"
```

The SDK automatically reads from `CHONKIE_API_KEY` if no key is provided explicitly.

## API Overview

### Chunking API

Perfect for developers who need:
- Text preprocessing for RAG pipelines
- Embedding generation at scale
- Flexible chunking strategies
- Direct integration with existing systems

**Popular Endpoints:**
- [Token Chunker](/api-reference/chunking/token-chunker) - Fast, simple chunking

## Common HTTP Status Codes

| Code | Meaning | Action |
|------|---------|--------|
| **200** | Success | Request completed successfully |
| **400** | Bad Request | Check request parameters |
| **401** | Unauthorized | Verify API key is correct |
| **404** | Not Found | Resource doesn't exist |
| **429** | Rate Limited | Slow down requests or upgrade plan |
| **500** | Server Error | Retry with exponential backoff |

## Best Practices

<Tip>
  **Cache API responses** when possible to reduce latency and stay within rate limits.
</Tip>

<Tip>
  **Use appropriate chunk sizes** based on your LLM's context window. Common sizes: 512 for GPT-3.5, 1024 for GPT-4.
</Tip>

<Tip>
  **Add overlap** between chunks (10-20%) to avoid losing information at boundaries.
</Tip>

<Tip>
  **Choose the right chunker** for your content type. Use Semantic for varied topics, Code for source code, Token for general text.
</Tip>

<Warning>
  **Never commit API keys** to version control. Use environment variables or secrets management.
</Warning>
