The **Vision** pipeline stage extracts text from images and PDF files using OCR, sitting between the Fetcher and Chef stages in the CHOMP pipeline. Currently powered by Mistral's OCR API.

## Installation

```bash
pip install chonkie[mistral]
```

Or use the vision extras group:

```bash
pip install chonkie[vision]
```

<Info>
  You need a Mistral API key. Set the `MISTRAL_API_KEY` environment variable or pass it directly.
</Info>

## Pipeline Usage

Use `.see_with()` to add OCR to your pipeline:

```python
from chonkie import Pipeline

# Process a PDF with OCR and chunk it
doc = (Pipeline()
    .fetch_from("file", path="document.pdf")
    .see_with("mistral")
    .chunk_with("recursive", chunk_size=512)
    .run())

print(f"Extracted {len(doc.chunks)} chunks from PDF")
```

### Parameters

<ParamField path="model" type="str" default="mistral-ocr-latest">
  The Mistral OCR model to use.
</ParamField>

<ParamField path="api_key" type="Optional[str]" default="None">
  Mistral API key. Falls back to the `MISTRAL_API_KEY` environment variable.
</ParamField>

```python
# Custom model and explicit API key
doc = (Pipeline()
    .fetch_from("file", path="scan.pdf")
    .see_with("mistral", model="mistral-ocr-latest", api_key="sk-...")
    .chunk_with("recursive", chunk_size=1024)
    .run())
```

## How It Works

The vision stage transforms the pipeline data flow:

```
Fetcher (file paths) â†’ Vision (extracted text) â†’ Chef (Documents) â†’ Chunker â†’ ...
```

1. The fetcher provides file path(s)
2. Vision sends images/PDFs to the OCR API and returns markdown text
3. The chef (default: TextChef) wraps the text into Document objects
4. Chunking proceeds as normal

## Supported File Types

| Type | Extensions |
|------|-----------|
| Images | `.png`, `.jpg`, `.jpeg`, `.gif`, `.bmp`, `.webp`, `.tiff`, `.tif` |
| Documents | `.pdf` |

## Examples

### OCR + RAG Pipeline

Build a complete pipeline from scanned documents to vector database:

```python
from chonkie import Pipeline

docs = (Pipeline()
    .fetch_from("file", dir="./scanned_docs", ext=[".pdf", ".png"])
    .see_with("mistral")
    .chunk_with("recursive", chunk_size=1024)
    .refine_with("overlap", context_size=100)
    .store_in("qdrant", collection_name="scanned_documents")
    .run())

print(f"Ingested {len(docs)} documents")
```

### OCR + Semantic Chunking

Use semantic chunking on OCR output for intelligent retrieval boundaries:

```python
from chonkie import Pipeline

doc = (Pipeline()
    .fetch_from("file", path="textbook_chapter.pdf")
    .see_with("mistral")
    .chunk_with("semantic", threshold=0.8, chunk_size=1024)
    .refine_with("embedding", model="text-embedding-3-small")
    .export_with("json", file="textbook_chunks.json")
    .run())
```

### OCR + Markdown Processing

Chain vision with the markdown chef for structural parsing:

```python
from chonkie import Pipeline

doc = (Pipeline()
    .fetch_from("file", path="formatted_report.pdf")
    .see_with("mistral")
    .process_with("markdown")
    .chunk_with("recursive", chunk_size=512)
    .run())

# Access markdown metadata
print(f"Found {len(doc.tables)} tables")
print(f"Found {len(doc.code)} code blocks")
```

## Standalone Usage

You can also use `MistralOCR` directly without the pipeline:

```python
from chonkie import MistralOCR

ocr = MistralOCR()

# Single file
text = ocr.process("research_paper.pdf")
print(text)

# Multiple files
texts = ocr.process_batch(["page1.png", "page2.png"])

# Async
import asyncio
text = asyncio.run(ocr.aprocess("document.pdf"))
```

## Creating Custom Vision Components

Extend `BaseVision` and register with `@vision` to create your own:

```python
from chonkie.vision import BaseVision
from chonkie.pipeline import vision

@vision("my_ocr")
class MyOCR(BaseVision):
    def __init__(self, **kwargs):
        # Initialize your OCR backend
        pass

    def process(self, file_path):
        # Extract text and return as string
        return "extracted text"

# Use in pipeline
doc = (Pipeline()
    .fetch_from("file", path="doc.pdf")
    .see_with("my_ocr")
    .chunk_with("recursive")
    .run())
```

## Notes

- OCR quality depends on image resolution and clarity
- Large PDFs are processed page-by-page and concatenated with double newlines
- The extracted text is returned as markdown, making it compatible with the `MarkdownChef` for further structural parsing
- API calls are synchronous by default; the pipeline's `arun()` uses async execution
